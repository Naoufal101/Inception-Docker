Inception: Docker Infrastructure

Description

The goal of this project is to broaden the knowledge of system administration by using Docker. This project involves setting up a small infrastructure composed of different services following specific rules for performance and security. The infrastructure is built using Docker Compose and includes a WordPress website, a MariaDB database, and NGINX server, along with extra monitoring tools.

Instructions

Build & Launch

Use the Makefile to manage the lifecycle of the application.

make

Creates the necessary volume directories (~/data) and builds the Docker images.

make up

Builds and starts the cluster in the foreground. You will see live logs from all containers.

make up-d

Builds and starts the cluster in detached mode (background). The terminal remains free for other commands.

Management

make stop

Stops the running containers without removing them.

make start

Starts existing containers that were previously stopped.

make down

Stops and removes the containers and the network. Data volumes are preserved.

Reset & Cleaning

make fclean

⚠️ Deep Clean: Stops containers, removes images, volumes, and deletes the local data folders.

make re

Performs a full reset (fclean) and then rebuilds the project from scratch.

Project Architecture Choices

This project utilizes Docker to containerize services. Here is the technical breakdown:

Virtual Machines vs Docker

VMs emulate an entire hardware stack and run a full OS. They are heavy and slow to boot.

Docker shares the host's Linux Kernel and only isolates the Application layer. It is lightweight and starts instantly.

Secrets vs Environment Variables

Env Variables are stored as plain text in the container config. Anyone with docker inspect can see them.

Secrets are encrypted and only mounted into the container's file system at runtime. They are more secure.

Docker Network vs Host Network

Docker Network creates an isolated virtual bridge. Containers talk via DNS names.

Host Network removes isolation. The container shares the exact IP and Port space of the host machine.

Docker Volumes vs Bind Mounts

Volumes are managed by Docker (in /var/lib/docker). They are the preferred way to persist data.

Bind Mounts link a specific folder on your Host to the Container.
